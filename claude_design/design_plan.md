# More Tea Community - Design Plan

## High-Level Overview

More Tea Community is a tea-focused community group based at UVIC ECS that meets weekly. The goal is a simple, warm, welcoming website where people can:

- Learn about the community
- See upcoming tea gathering events on a calendar
- Join the Google Group mailing list
- Follow on Instagram
- Browse past events (Archive)

The site will live at `alimorty.github.io/more_tea_community` and be hosted for free via GitHub Pages. It is a fully static site - no server, no database, no login system.

---

## Pages

| Page | Purpose |
|------|---------|
| Home | Hero banner, calendar, sidebar signup, about snippet |
| Calendar | Full calendar view of events |
| Connect | Google Group signup form, Instagram link, contact info |
| About | Story of the community, photos, who runs it |
| Archive | Past events and photos |

---

## Visual Design

- **Color palette**: warm cream background, dark brown text, gold accents (pulled from the logo)
- **Font**: a serif or semi-serif for headings (warm, classic feel), clean sans-serif for body
- **Logo**: `Logo.png` appears in the top-left navbar
- **Tone**: cozy, welcoming, community-first

## Technical Plan

### Hosting

- GitHub Pages, served from the `docs/` folder on the `main` branch
- No build step. Push HTML/CSS/JS files and they go live.
- URL: `https://alimorty.github.io/more_tea_community`

### File Structure

```
docs/
  index.html          # Home page
  calendar.html       # Calendar page
  connect.html        # Connect / signup page
  about.html          # About page
  archive.html        # Archive page
  css/
    style.css         # All shared styles
  js/
    calendar.js       # Calendar logic
  images/
    logo.png          # Site logo
    hero.png          # Hero banner image (to be generated via Gemini)
    (other images as needed)
```

### Technology

- **HTML5** - semantic markup, one file per page
- **CSS3** - custom properties (variables) for colors, flexbox/grid for layout, responsive design with media queries
- **Vanilla JavaScript** - calendar rendering only, no framework needed
- **No npm, no webpack, no React** - open the HTML file and it works

### Calendar

- Events are stored as a plain JavaScript array in `js/calendar.js`
- The calendar renders a monthly grid, highlighting days with events
- Clicking an event shows a small popup with details (name, location, time, description)
- To add a new event, edit the array in `calendar.js` - no database needed

### Sidebar Signup

- The "Join Our Community" form points to the Google Group join URL or a Google Form
- The Instagram button links directly to the Instagram profile
- No backend required

### Responsiveness

- The layout stacks vertically on mobile (sidebar moves below main content)
- Navigation collapses to a hamburger menu on small screens
